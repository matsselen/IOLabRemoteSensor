int _system_pre_init ( void  )

System pre-init function that is the very first code to be run. Overrides the function provided by the TI library.

Note
Only microcontroller registers should be accessed. Any use of variables will probably be overwritten.

From TI: _system_pre_init() is called in the C/C++ startup routine (_c_int100() in boot.c) and provides a mechanism for the user to insert application specific low level initialization instructions prior to calling main(). The return value of _system_pre_init() is used to determine whether or not C/C++ global data initialization will be performed (return value of 0 to bypass C/C++ auto-initialization).

PLEASE NOTE THAT BYPASSING THE C/C++ AUTO-INITIALIZATION ROUTINE MAY RESULT IN PROGRAM FAILURE.

The linker will include the updated version if it is linked in prior to linking with the C/C++ runtime library (rts430.lib).

Return values
0- if variable auto-init code should NOT be run
1- if variable auto-init code should be run
